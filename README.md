## 昇思MindSpore技术公开课

### 课程亮点

昇思 MindSpore 技术公开课是面向 AI 开发者的品牌课程，覆盖深度学习到模型训练部署全链路，以 “理论 + 实践” 模式提供完备学习资源，由浅入深解读AI 技术。

- **结构化课程**：全链路分层设计，从入门到进阶形成完整学习路径，适配不同阶段需求。
- **探究前沿**：覆盖大模型预训练、微调、推理优化等前沿技术，同步行业实践。
- **专家解读**：昇思团队及领域专家参与，拆解核心原理与落地经验。
- **动手实践**：知识点配套可运行代码与操作指南，结合硬件场景实操。
- **开源共享**：课程资源（PPT、文档、代码）开源开放，支持复用与协作。

## 贡献与反馈

欢迎各位开发者在对应项目仓中，通过 Issue 提交建议或 bug 反馈，也可直接发起 PR 进行Bug修复或代码贡献（提交前请参考仓库贡献规范，由项目仓对应Committer完成评审合入），你的每一份参与都能让项目仓更加完善。

### Issue提交规范
- **Bug 反馈（Bug Report）**：请在标题注明「Bug」，正文说明复现步骤（含环境配置、输入参数）、预期结果与实际报错信息（附截图或日志），并标注使用的版本名；
- **功能建议（Feature Request）**：请在标题注明「Feature」，正文清晰描述需求场景（例：“希望补充XXX模型的微调教程”）及预期价值，便于评估优先级；
- **疑问咨询（Question）**：请在标题注明「Question」，先检索历史 Issue 确认无重复后，再描述具体疑惑或卡点（如 “XXX课节模型构建部分代码运行报错，如何排查？”），建议附相关代码或终端输出。

### PR提交规范
- **内容要求**：提交代码需符合Pylint规范，在关键代码片段补充注释，代码需通过自验（附运行成功截图），文档更新保证Markdown格式规范。
- **提交格式**：PR 标题以「模块名：具体修改」命名，正文说明修改动机、核心变更点，若关联 Issue 请备注「Fix #XXX」；
- **审核流程**：提交后将自动触发代码规范检查，通过后将由Committer审核，审核通过后合并至主分支，贡献者将被列入致谢名单。

## Committer

### 核心职责

Committer是负责代码提交审核、维护仓库质量的核心维护者，主要职责包括：

1. 管理项目仓的版本发布
2. 审核并合并社区贡献的代码 PR
3. 把控项目代码风格与功能稳定性
4. 响应社区规范冲突并解决仓库冲突、响应 issue 并推动问题修复
5. 保障该项目中资源（代码、文档、课件等）的持续更新与可用性，确保学习者获得准确、优质的实践内容

项目仓的所有PR均需在Committer审核通过后方能合入。
